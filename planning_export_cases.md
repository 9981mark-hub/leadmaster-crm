# 📄 케이스 내보내기(Export) 기능 고도화 기획서

## 1. 개요 (Overview)
현재 케이스 목록 페이지 상단에 위치한 **"특수 케이스 내보내기"** 버튼은 클릭 시 즉각적으로 전체(또는 특정 조건의) 데이터가 CSV로 다운로드되는 구조입니다. 
데이터가 쌓일수록 불필요한 데이터까지 한 번에 다운로드되어 비효율이 발생하며, 사용자가 원하는 특정 조건(월별, 상태별 등)의 데이터만 선별하여 추출하기 어렵습니다.
이에 따라 **다운로드 전 조건 설정 모달(Modal)**을 제공하여, 사용자가 원하는 맞춤형 데이터를 쉽게 추출할 수 있도록 기능을 고도화하고자 합니다.

---

## 2. AS-IS vs TO-BE

| 구분 | AS-IS (현재) | TO-BE (변경 후) |
|---|---|---|
| **동작 방식** | 버튼 클릭 즉시 다운로드 실행 (One-click) | 버튼 클릭 시 **[내보내기 옵션 모달]** 팝업 노출 |
| **추출 대상** | 고정된 특수 케이스 기준 전체 | 1차 상태별, 월별, 기간별, 대상별 상세 필터링 가능 |
| **유연성** | 매우 낮음 | 매우 높음 (원하는 조건 조합 가능) |

## 3. UI/UX 및 세부 기능 기획 (기능 요구사항)

버튼의 명칭을 **"데이터 내보내기"** (또는 기존 유지: "특수 케이스 내보내기" ➔ 클릭 시 모달 노출)로 변경하고, 클릭 시 나타나는 **모달 창**에 다음과 같은 옵션을 구성합니다.

### 3.1. 추출 대상 범위 설정 (Target Scope)
사용자가 다운로드할 기본 데이터의 범위를 선택합니다.
*   🔘 **전체 케이스**: 시스템에 등록된 모든 케이스 대상
*   🔘 **특수 케이스**: (기존 버튼 기능 유지) 별도 관리 대상인 특수 케이스만 대상
*   🔘 **현재 목록**: 현재 화면에서 검색 및 필터링된 결과만 대상 (Pagination 무시하고 검색된 전체)

### 3.2. 1차 상태(Status) 필터링
케이스의 진행 상태(1차 상태)를 다중 선택하여 내보낼 수 있습니다.
*   ✅ **전체 상태** (기본값)
*   ⬜️ 개별 상태 선택 (Checkbox 형태): `접수`, `부재`, `상담완료`, `진행불가`, `계약완료` 등 시스템 내 1차 상태 동적 렌더링

### 3.3. 기간 / 월별 필터링 (Date Range)
데이터의 **등록일(createdAt)** 또는 **계약일(contractAt)**을 기준으로 기간을 설정합니다.
*   **기준일 선택**: 
    *   🔘 등록일 기준 (기본값)
    *   🔘 상태변경일 기준
    *   🔘 계약일 기준
*   **기간 설정 방식**:
    *   🔘 **전체 기간** (모든 데이터)
    *   🔘 **특정 월 선택**: `[ 2026년 ]` `[ 4월 ]` (드롭다운으로 연/월 선택 시 해당 월 1일~말일 자동 설정)
    *   🔘 **직접 지정**: `YYYY-MM-DD` ~ `YYYY-MM-DD` (Date Picker 연동)

### 3.4. 추출 항목 (Column) 설정 [Optional - 확장 기능]
필요에 따라 엑셀(CSV)에 포함될 열(Column)을 가감할 수 있는 기능입니다.
*   ✅ **기본 양식**: 필수적인 개인정보 및 상태 정보만 포함
*   ✅ **전체 항목**: 모든 DB 항목 포함
*   *(추후 고도화 시, 체크박스로 원하는 컬럼만 커스텀 선택 가능하도록 확장)*

## 5. 기대 효과
1.  **휴먼 에러 방지**: 실수로 버튼을 눌러 불필요한 대용량 다운로드가 발생하는 것을 방지 (트래픽 및 리소스 절약).
2.  **업무 효율성 증대**: 월별 보고서, 특정 상태(예: 진행불가 사유 파악, 계약완료 건 정산)의 통계 산출을 위해 엑셀에서 2차 가공할 필요 없이, 시스템 내에서 바로 원하는 데이터만 추출 가능.
3.  **보안 강화**: 무분별한 전체 데이터 유출(다운로드)을 줄이고, 꼭 필요한 범위의 데이터만 취급하는 환경 조성.

## 6. 개발 진행 (Next Step) 제안

1.  `ExportCaseModal.tsx` 컴포넌트 신규 생성

3.  모달 내에서 선택된 상태/기간 옵션 조합을 기반으로 데이터를 필터링하는 CSV Export 로직(`utils.ts` 등에 구현된 기존 로직 확장) 연동
